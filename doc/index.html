Yet Another Menu

Download Version 0.1, 4kB Source Repo on github

Accessible, responsive dropdown menu

Works with every* device

yam.js works on every screen resolution and uses Modernizr to check, if the device is supporting touch gestures.

The lib repositions the submenues on resize, but no worry: Ben Alman's debounce-plugin makes sure, no one's CPU is getting killed.

* Ok... I haven't tested it on every device, but there are no special browser-requirements needed.

Can be used in real life Projects

yam.js was build for a real life project for a client. If you are using yam.js, please tell me! Also you can ask for feature-requests or give some general feedback.

I've got the lib tested at least on this Browsers:

  • Internet Explorer: 7 8 9 10
  • Chrome
  • Firefox
  • Safari
  • Opera

Documentation

You can manipulate the way, yam.js processes your custom markup and how the menu is displayed. All configurations are passed when calling yam on an object.

Calling yam.js with the following options. (The displayed ones are the default values) :

$(document.ready(function(){
  $("#menu").yam({
    hover_delay: 1000,
    window_min_width: 640,
    remember_menu_state: false,
    hover_animations: {
      "in": 100,
      "out": 100
    },
    layout_vertical_at: 3,
    selectors: [
      ['self', '> li'],
      ['> ul, > ol', '> li']
    ],
    callbacks: {
      mouseenter: null,
      mouseleave: null,
      click: null
    },
    click_events: 'click, touchend',
    is_active: function(element) {
      return element.hasClass('active');
  });
}));

All config options

hover_delay

The time, the user has to reenter a list element, before it's children will be hidden.

window_min_width

When the window is smaller then the min_width, the menu will be hidden.

remember_menu_state

Horizontal menus can be remembered. The last horizontal menu in one level will remain open.

hover_animations

Set the animation duration for menu display and hide.

layout_vertical_at

Set the first level for vertical dropdowns. All deeper menu levels will also be vertical.

selectors

If your markup isn't just a plain list, because your CMS renders crapy code, you can set the selector. Each array entry represents one menu level. All deeper menu levels will use the last selector.

callbacks

Register your callback function, if the mouse enters, leaves or clicks on an element.

click_events

Set the events, yam.js should react on.

is_active

Set the function to check, if an element is active by default.
